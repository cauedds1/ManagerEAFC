Quero criar um sistema web chamado FC Career Manager para gerenciar o modo carreira do EA FC 26. O problema que resolvo: o jogo salva estatísticas de forma muito pobre, então quero um lugar externo para registrar partidas, jogadores e evolução da temporada com imersão máxima.
O sistema deve ser 100% em Português do Brasil (PT-BR) — todos os textos, labels, botões, mensagens de erro, placeholders, tooltips e qualquer string visível ao usuário deve estar em PT-BR. Nenhuma palavra em inglês deve aparecer na interface.

FONTE DE DADOS — Leia com atenção
Todos os dados de jogadores e clubes vêm exclusivamente desta API:
Base URL: https://api.msmc.cc/api/eafc
Endpoints que você vai usar:
# Todos os jogadores do FC26
GET https://api.msmc.cc/api/eafc/players?game=fc26

# Todos os clubes masculinos do FC26
GET https://api.msmc.cc/api/eafc/clubs?gender=m&game=fc26

# Todas as ligas masculinas do FC26
GET https://api.msmc.cc/api/eafc/leagues?gender=m&game=fc26

# Filtrar jogadores por clube
GET https://api.msmc.cc/api/eafc/players?game=fc26&team=Barcelona

CONCEITO CENTRAL — Temas dinâmicos por clube
Ao selecionar um clube, o sistema inteiro muda visualmente para as cores oficiais daquele clube. Isso é o coração do produto — deve ser fluido, bonito e imersivo.

FASE 1 — O que construir agora
Implemente apenas o seguinte. Nada além disso.
1. Tela de Seleção de Clube

Buscar todos os clubes via GET /clubs?gender=m&game=fc26
Grid visual com escudo + nome + liga de cada clube
Campo de busca por nome do clube em tempo real — placeholder: "Buscar clube..."
Filtro por liga — label: "Filtrar por liga"
Ao clicar em um clube, salvar a seleção e entrar no dashboard
Animação suave de entrada dos cards

2. Sistema de Temas Dinâmicos

Ao selecionar o clube, aplicar imediatamente as cores no CSS via variáveis
Transição animada entre temas (300ms ease)
Todo o sistema — sidebar, botões, headers, cards, gradientes — deve usar var(--club-primary) e var(--club-secondary)
Nenhuma cor hardcoded fora do mapeamento de clubes

3. Dashboard Principal

Header com escudo + nome do clube + temporada atual (ex: 2025/26)
3 cards de estatísticas placeholder com os seguintes textos em PT-BR:

"Partidas" — ícone de bola, valor: 0
"Elenco" — ícone de pessoas, valor: 0
"Transferências" — ícone de setas, valor: 0


Seção "Próximas funcionalidades" mostrando o roadmap visual do que vem nas próximas fases
Botão "Trocar de clube" que volta para a tela de seleção

4. Persistência

Salvar clube selecionado e temporada no localStorage
Ao reabrir o app, carregar direto no dashboard do clube salvo sem passar pela tela de seleção novamente

DESIGN — Regras obrigatórias

Tema base: escuro (background #0a0a0a ou similar)
Tipografia: Inter (Google Fonts)
Cards: glassmorphism — background: rgba(255,255,255,0.05), backdrop-filter: blur(10px), borda sutil com a cor primária do clube
Animações: transições suaves em hover, entrada com fade+slide, troca de tema com cross-fade
Responsivo: funcionar bem em desktop e mobile
Qualidade: deve parecer um produto comercial real, não um projeto de faculdade


STACK

React + TypeScript
Tailwind CSS
localStorage para persistência (por agora)
color-thief-browser para extração de cores automática
Sem backend por enquanto — tudo client-side
